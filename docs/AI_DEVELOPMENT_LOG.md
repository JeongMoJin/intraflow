# AI Development Log

이 문서는 AI 도구 사용 자체를 강조하기보다, 요구사항을 해석하고 결과물을 검증하는 과정을 기록하기 위한 문서입니다.

## AI 도구 사용 범위

- 포트폴리오 요구사항을 기능 단위로 분해
- ASP.NET Core API 구조와 React 관리자 화면 초안 작성
- README와 보안/면접 문서 초안 작성
- 테스트 케이스 후보 도출

## 사람이 검토한 기준

- 실행 가능한가: `dotnet build`, `dotnet test`, `npm run build`
- 채용공고 기술스택과 직접 연결되는가
- 과한 추상화로 설명이 어려워지지 않는가
- 실제 secret이나 민감정보가 포함되지 않았는가
- 감사 로그에 개인정보가 과도하게 남지 않는가
- 2년차 후반 개발자가 설명해도 자연스러운 범위인가

## AI에 넣으면 안 되는 정보

- 실제 회사 코드
- 운영 DB 데이터
- Microsoft Graph client secret
- access token, refresh token, cookie
- 사내 구성도, 계정 목록, 고객 개인정보

## 생성 코드 검증 체크리스트

- API별 권한이 의도대로 제한되는지 확인
- 전자결재 승인/반려 시 AuditLog 저장 확인
- ERP Sync 성공/실패 케이스 확인
- CSV Import 실패 사유가 LegacyImportLog에 남는지 확인
- Manager/Employee 권한에서 이메일 마스킹 확인
- README 실행 명령이 실제 폴더 구조와 일치하는지 확인

## AI 결과물의 한계

AI는 실행 환경, 패키지 버전, 보안 정책을 자동으로 보장하지 않습니다. 따라서 패키지 취약점 경고, 빌드 오류, 타입 오류, 권한 누락은 사람이 확인해야 합니다.

## 검증 사례

- SQLite 하위 의존성 취약점 경고를 확인하고 최신 SQLitePCLRaw bundle을 직접 참조했습니다.
- .NET 10 SDK 템플릿이 `net10.0`으로 생성된 것을 `net8.0`으로 조정했습니다.
- Vite 생성 명령이 React 템플릿으로 생성되지 않은 것을 확인하고 React/DevExtreme 의존성과 설정을 직접 보완했습니다.

## 결론

이 프로젝트에서 AI는 구현 속도를 높이는 보조 도구로 사용했습니다. 최종 판단 기준은 요구사항 충족, 보안, 유지보수성, 실행 가능성이며, 결과물은 빌드와 테스트로 검증했습니다.
